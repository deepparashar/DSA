Here's a summary about Binary tree:-

-What is Binary Tree?
Binary Tree is a non-linear hierarchical Data Structure which stores data in a form of tree
in a parent-child relationship like a family tree.In a binary tree each node has at most 2 children 
and a node with no children is called a leaf node.

    Here's a Structure:-
               
                  |
                 Root  (Top most Root also a Parent node)    
           |              | 
       Left child        Right child     (left and right node of root as a Child node)
     |          |      |          |
    left      right   left       right 
   (leaf)     (leaf)  (leaf)      (leaf)

    
- This is used to store data more efficiently and fast.It organizes data in a hierarchical manner which makes
  searching, insertion and deletion operation much faster as compared to array.
 
 

 -Types of Binary tree:-
 
 1) Full binary tree: In full binary tree each node has either 0 or 2 childern
                       No node in full binary tree has 1 child.

 2) Complete binary tree: In complete binary tree every level is completely filled 
                        except last level.In last level adding Left is possible.

 3) Perfect binary tree: All nodes are at same level. it is both full and complete.

 4) Degenerate Tree: A tree where every internal node has only one child. This structure
                      essentially behaves like a linked list in terms of performance.

 5) Skewed Binary Tree: A special case of a degenerate tree where all nodes are either
                        on the left (Left-skewed) or all on the right (Right-skewed).      

                                                               


 -Traversal of Binary tree:

1) DFS (Depth-first search): This traversal implemented using recursion or via Stack (LIFO)
                             these traversal goes in depth of tree.

   A) Pre-order traversal: Root -> left -> right
      Ex: 1 -> 2 -> 3            

   B) In-order traversal: left -> Root -> right
      Ex: 2 -> 1 -> 3

   C) Post-order traversal: right -> left -> root
       Ex: 3 -> 2 -> 1

 2) BFS  (Breadth-first search) / Level order traversal: Visits nodes level by level, from
                                                         left -> right using Queue (FIFO)
